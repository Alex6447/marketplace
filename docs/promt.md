## Стек проекта (из plan.md, кратко)

- Backend: Python 3.12 + FastAPI (тонкий образ), отдельный GPU/CPU worker (тяжёлый образ).
- Очередь: Redis + Celery. БД: PostgreSQL. Файлы: MinIO (S3). Тулинг: uv + ruff, pnpm.
- Frontend: React + TypeScript + Vite, TanStack Query, shadcn/ui.
- Провайдер-абстракции `LLMProvider` / `ImageProvider` (hosted ⇄ local), editing-модели + композитинг, текст через Playwright, прогресс через SSE.
- Стратегия: **API-first** (hosted-модели для MVP), локальные модели — отдельный этап.

## Что делать

1. Открой раздел 7 «Поэтапный план работ» в [docs/plan.md](plan.md).
2. Найди **первую невыполненную задачу** (чекбокс `- [ ]`), идя сверху вниз по этапам.
3. Начни выполнять её. Соблюдай глобальные правила из CLAUDE.md (минимальное воздействие, не трогать продовые данные, проверять разрушительные действия).
4. Перед каждой задачей сверяйся с планом и принципами из разделов 1–6, чтобы реализация соответствовала архитектуре.
5. Работай последовательно: доводи задачу до рабочего состояния, прежде чем переходить к следующей.
6. Не выполняй разом весь план — двигайся по задачам, чтобы прогресс был обозримым. Если задача крупная, разбей её и сообщи план перед реализацией.

## ВАЖНО: отмечай прогресс в plan.md

- **Как только задача в разделе 7 выполнена и проверена — поменяй её чекбокс с `- [ ]` на `- [x]` прямо в [docs/plan.md](plan.md).**
- Это нужно, чтобы в следующей сессии (при повторном запуске этого промта) ты начал с другой, ещё не сделанной задачи, а не повторял уже готовую.
- Если задача выполнена лишь частично — не помечай её как `[x]`; вместо этого добавь короткую заметку о текущем состоянии рядом с пунктом.
- Коммить изменения по-русски осмысленными коммитами. **Не пушь** без отдельной просьбы.
