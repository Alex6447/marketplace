# Выбор хостинга для размещения (модели — по API, без GPU)

Документ по выбору площадки для размещения системы генерации карточек при сценарии
**«все модели — внешние API»** (hosted-режим, см. [plan.md](plan.md) 4.1): на сервере
нет GPU и тяжёлых ML-весов, считают внешние сервисы. Сравнение по зонам **РФ / Европа /
Китай**, критерий — **цена/качество** + помесячная стоимость.

> Цены — ориентировочные, на июнь 2026 (источники внизу). Курсы и тарифы плывут —
> перед закупкой сверять на сайтах провайдеров.

---

## 1. Что хостим

Модели по API → GPU не нужен. Один VPS под `docker-compose`:

- `api` (тонкий FastAPI) + `worker` (Celery; стадии [4]–[9], текст [6] через
  Playwright/Chromium — нужен ~1–2 ГБ RAM под браузер);
- `redis` (брокер/очередь), `postgres` (метаданные), `minio` (S3-хранилище картинок),
  `nginx` + статика фронтенда.

Нагрузка в основном **I/O-bound** (ожидание внешних API), поэтому требования скромные.

| Конфигурация | vCPU / RAM / диск | Когда |
|---|---|---|
| **Базовая (MVP)** | 4 vCPU / 8 ГБ / 160 ГБ NVMe | старт, низкий/средний поток |
| Рост | 8 vCPU / 16 ГБ / 240 ГБ | рост потока; вынести Postgres/Redis/S3 в managed |

---

## 2. ⚠️ Главное ограничение: доступность модельных API по зонам

Проект завязан на **Claude API** (стадии [2]/[3]/[9]) и **Gemini/Flux** (стадия [5]).
Их доступность радикально зависит от зоны размещения:

| Зона | Claude / Gemini | Следствие |
|---|---|---|
| **Европа** | ✅ работают официально | топовое качество edit-моделей «из коробки» |
| **РФ** | ❌ заблокированы Anthropic/Google | либо egress-прокси (хрупко, риск нарушения ToS), либо переход на РФ-модели: **YandexART/Kandinsky** (картинки) + **YandexGPT/GigaChat** (LLM) — качество «сохранения товара» ниже |
| **Китай** | ❌ заблокированы + Great Firewall | использовать китайские API: **Qwen-Image-Edit** (Alibaba DashScope — уже заложен в архитектуру), **Tongyi/Hunyuan**; LLM — Qwen/ERNIE/GLM |

Благодаря провайдер-абстракциям (`ImageProvider`/`LLMProvider`) смена набора моделей под
зону — это конфиг/новая реализация провайдера, без переписывания пайплайна.

### 🇨🇳 Китай

| Провайдер | Конфиг 4/8 | Цена/мес |

## 4. Стоимость моделей по API в месяц

По сути одинакова для всех зон (различается провайдер модели). Картинки [5] —
основная статья расхода. Допущение: 1 комплект = 6 карточек, ×1.4 на правки/фидбэк.

| Объём | Картинки (Gemini ~$0.04/шт) | LLM (Claude Sonnet) | **Итого API/мес** |
|---|---|---|---|
| 100 комплектов/мес | ~840 img → ~$34 | ~$15 | **≈ $50** |
| 500 комплектов/мес | ~4 200 img → ~$170 | ~$75 | **≈ $245** |

⚠️ Текущий дефолт LLM — **Claude Opus** (дорогой). Для стадий [2]/[3]/[9] рекомендуется
**Sonnet/Haiku**: качества достаточно, LLM-часть дешевле в ~5–7×. Учёт фактической
стоимости — уже реализован (эндпоинт `/card-sets/{id}/cost`, `observability/cost.py`).

## 5. Итоговый расчёт «под ключ» и рекомендация

| Зона | Хостинг/мес | API/мес (100 / 500 компл.) | Качество edit-моделей | Примечание |
|---|---|---|---|---|
| **🇪🇺 Европа (Hetzner)** | €20–35 (~$22–38) | $50 / $245 | ★★★ (Flux Kontext/Gemini) | **лучший цена/качество**, ноль ML-ops |
| 🇷🇺 РФ (Timeweb/Selectel) | ~$35–90 | сопоставимо, но на РФ-моделях | ★★ (YandexART/Kandinsky) | 152-ФЗ; Claude/Gemini только через прокси |
| 🇨🇳 Китай (Alibaba HK) | ~$50–90 | сопоставимо (DashScope) | ★★–★★★ (Qwen-Image-Edit) | ICP для материка; HK проще |

### Вывод

> **Базовый выбор: Hetzner (Германия/Финляндия), CPX31 → CPX41, + Claude (Sonnet) +
> Gemini Flash Image.** Хостинг **€20–35/мес**, модели **$50–245/мес** по объёму.
> Лучшее качество сохранения товара, минимальная цена железа, провайдеры уже в коде.

- Требуется **юрисдикция данных РФ** → Timeweb/Selectel/Yandex Cloud, но менять модели на
  YandexART/GigaChat (ниже качество «сохранения товара») либо ставить egress-прокси.
- Нужен **Китай** → Alibaba Cloud (Гонконг) + **Qwen-Image-Edit** через DashScope
  (переключается через `ImageProvider`, уже заложено архитектурно).
- При любом росте: вынести **Postgres** и **S3** в managed-сервисы провайдера (надёжные
  бэкапы), `worker` масштабировать горизонтально (см. Этап 7 — профили docker-compose).
